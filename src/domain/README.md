# domain

Regras de negócio puras: entidades, value objects, regras de validação e as interfaces
(portas) que descrevem o que a camada de infraestrutura precisa fornecer.

Regra: nada aqui importa de `application/`, `infrastructure/` ou `interface/`. Esta camada
não conhece frameworks, bibliotecas de banco de dados ou detalhes técnicos externos.

Veja [docs/architecture.md](../../docs/architecture.md) para o racional completo.
